00

개요 & 비전

"번역을 잘하는 도구"가 아니라 "번역 품질을 조직 차원에서 일관되게 유지·축적하는 시스템". 사람의 판단(감수·맥락)과 AI의 속도를 결합하는 것이 ATLAS의 핵심 철학입니다.

🎯 미션

AI로 번역을 가속하되, 용어·어투·맥락의 일관성을 사람이 통제할 수 있는 구조를 제공한다.

💡 차별점

단순 MT가 아닌 화자·맥락 인지형 번역. 기존 번역 문서를 AI가 역분석해 자산(TM/용어/화자)으로 자동 DB화.

🏢 운영 단위

법인 → 스튜디오 → 프로덕트의 3계층으로 분리 관리하며, 멤버별 역할 기반 권한(RBAC)을 부여한다.

01

문제 정의 & 타깃 사용자

현장의 로컬라이제이션은 "도구 부재"보다 "일관성·자산화·협업"의 문제가 더 큽니다.

해결하려는 문제(Pain)

  • 용어 불일치 — 동일 용어가 챕터·번역가마다 다르게 번역됨
  • 어투/화자 붕괴 — 캐릭터 말투, 존댓말/반말 일관성 깨짐
  • 맥락 손실 — 앞뒤 내러티브를 모르고 번역 → 오역
  • 자산 휘발 — 과거 번역물이 파일로만 남아 재사용 불가
  • 감수 비효율 — LQA 기준이 사람 머릿속에만 존재
  • 조직 분리 부재 — 여러 타이틀/팀의 데이터 혼재

사용자 역할(Persona)

  • PM / 로컬 디렉터 — 프로젝트 세팅, 진척·품질 관리
  • 번역가(Translator) — 세그먼트 번역, 용어·맥락 참조
  • 감수자(Reviewer/LQA) — 품질 검수, 이슈 등록
  • 용어/내러티브 관리자 — 용어집·화자·세계관 정의
  • 스튜디오/법인 관리자 — 조직·멤버·권한·정책
  • 외부 협력사(Guest) — 제한된 범위 접근
02

핵심 기능 모듈

기존 요구사항을 8개 모듈로 정제했습니다. 각 모듈은 독립적으로 가치를 제공하면서도 AI 파이프라인과 자산 DB를 공유합니다.

01
CORE

AI 번역 엔진

  • 용어집·TM·화자·맥락을 프롬프트에 자동 주입한 컨텍스트 인지형 번역
  • 세그먼트 단위 / 문서 일괄(Batch) 번역
  • 대안 번역 제안 · 신뢰도(confidence) 표시
  • 모델 선택(품질↔속도) 및 다국어 동시 번역
02
ASSET

용어집(Glossary) 관리

  • 용어/번역/품사/도메인/금지어/DNT(번역 금지) 속성
  • 프로덕트별·언어쌍별 용어집, 상속/오버라이드
  • 번역 중 실시간 용어 매칭 & 위반 경고
  • 승인 워크플로우(제안→검토→확정)
03
CONTEXT

화자 & 어투 관리

  • 캐릭터 프로필(성격·나이·말투·관계도)
  • 화자별 어투 규칙(존댓말/반말, 1인칭, 어미 패턴)
  • 세그먼트에 화자 태깅 → 번역 시 어투 자동 반영
  • 화자 간 관계에 따른 호칭 일관성 관리
04
CONTEXT

내러티브 맥락 관리

  • 세계관/스토리 아크/씬(scene) 단위 맥락 노트
  • 세그먼트-씬 연결, 앞뒤 문맥 자동 참조 윈도우
  • 분기/선택지 등 비선형 서사 구조 표현
  • 스포일러/톤 가이드 등 번역 지침 첨부
05
CORE

문서 인제스트 & 자동 분석

  • 기존 번역 문서 업로드(xlsx·csv·xliff·json·docx)
  • AI가 화자 추정 · 용어 추출 · 맥락 분류 자동 수행
  • 원문/번역 정렬(alignment) → TM 유닛 생성
  • 검수 후 용어집·화자·TM으로 DB화
06
ASSET

번역 메모리(TM)

  • 정확/퍼지(fuzzy)/의미 기반(임베딩) 매칭
  • 매치율에 따른 자동 채움 & 재활용
  • 프로덕트·도메인별 TM 분리 및 우선순위
  • 중복·충돌 정리(메인터넌스) 도구
07
QA

LQA & 감수

  • 자동 QA: 용어 준수·어투·태그/플레이스홀더·길이·금지어
  • 분야 적합성 감수(해당 도메인 표현인지 판정)
  • 이슈 등록·심각도·코멘트·재작업 루프
  • 품질 점수(LQA score) 및 리포트
08
ADMIN

조직 · 멤버 · 권한

  • 법인 / 스튜디오 / 프로덕트 3계층 분리
  • 역할 기반 권한(RBAC) + 프로젝트별 할당
  • 초대·SSO·감사 로그(audit log)
  • 대시보드: 진척·품질·작업량 가시화
03

데이터·과금·접근 통제의 경계를 3계층으로 정의합니다. 모든 자산(용어/화자/TM/맥락)은 상속 가능하되 하위에서 오버라이드할 수 있습니다.

ATLAS의 AI는 두 갈래로 동작합니다 — (A) 신규 번역 생성, (B) 기존 문서 역분석·자산화.

05

번역 워크스페이스 (에디터)

CAT 도구 스타일의 3단 레이아웃 — 세그먼트 그리드(좌) · 번역 입력(중) · 컨텍스트 패널(우).

① 세그먼트 그리드

  • 원문/번역 행 단위 · 상태 색상
  • 화자·씬 태그 표시
  • 필터(미번역·이슈·화자별)
  • 일괄 선택·일괄 번역

② 번역 에디터

  • AI 제안 + 대안 후보
  • TM 매치 자동 채움
  • 인라인 용어 하이라이트
  • 태그/플레이스홀더 보호

③ 컨텍스트 패널

  • 현재 화자 프로필·어투
  • 관련 용어집 항목
  • 씬/내러티브 노트
  • QA 경고 · 감수 코멘트

번역 상태 흐름 (Status Workflow)

미번역→ AI 초벌→ 번역중→ 번역완료→ 감수중→ 반려/재작업⟲ 승인→ 배포/내보내기
06

자산 관리 상세 (용어 · 화자 · 맥락)

ATLAS의 핵심 가치는 "번역 자산의 구조화된 축적"입니다. 각 자산의 속성을 정의합니다.

📘 용어집 항목

  • 원어 / 번역어
  • 품사·도메인 분류
  • DNT (번역 금지 / 고유명사)
  • 금지 표현 블랙리스트
  • 용례·주석
  • 상태 제안/확정/폐기

🎭 화자 프로필

  • 이름·역할·성격
  • 어투 (존/반말, 어미)
  • 1인칭 / 호칭 규칙
  • 관계도 (타 화자 대상 말투)
  • 언어별 어투 변형
  • 예시 대사

🗺️ 내러티브 맥락

  • 세계관·설정 노트
  • 스토리 아크 / 씬
  • 분기·선택지 구조
  • 톤 가이드
  • 스포일러 주의 플래그
  • 세그먼트 연결
07

LQA & 감수 체계

"자동으로 잡을 수 있는 것은 자동으로, 판단이 필요한 것은 사람이." 2단계 품질 관문을 둡니다.

자동 QA 체크 (규칙 + AI)

  • 용어 준수 — 용어집 위반·금지어 검출
  • 어투 일관성 — 화자 규칙 대비 존/반말·어미
  • 태그·플레이스홀더 — {name}, 마크업 무결성
  • 길이·UI 제약 — 글자수/줄바꿈 한도
  • 누락·미번역 — 빈 세그먼트, 원문 잔류
  • 숫자·서식 — 단위·날짜·통화 일치

사람 감수 (LQA)

  • 분야 적합성 — 해당 도메인 실사용 표현인지
  • 맥락 정확성 — 내러티브 흐름상 오역 여부
  • 화자성 — 캐릭터다움 / 톤 유지
  • 이슈 등록 — 심각도(Critical~Minor)·카테고리
  • 재작업 루프 — 반려 → 수정 → 재검
  • 품질 점수화 — LQA 스코어·오류 유형 통계
감수 이슈는 MQM/LISA 류 표준 카테고리(정확성·유창성·용어·스타일·로케일)를 차용해 정량화하면, 번역가·벤더별 품질 비교와 개선 추적이 가능합니다.

09

기술 스택 제안

AI·벡터검색·실시간 협업·멀티테넌시를 고려한 권장 구성(확정 아님, 논의용).

프론트엔드

  • Next.js + TypeScript (App Router)
  • Tailwind CSS + 컴포넌트 시스템
  • 가상 스크롤(대량 세그먼트), 실시간 협업(WebSocket/CRDT)

백엔드 / 데이터

AI

  • Claude API — 분석/감수: claude-opus-4-8, 대량 번역: claude-sonnet-4-6
  • 임베딩 기반 TM/맥락 검색, 프롬프트 템플릿 관리
  • 토큰·비용 모니터링

운영 / 연동

  • SSO/OAuth, RBAC, 감사 로그
  • 가져오기/내보내기: XLIFF·CSV·XLSX·JSON + 게임 엔진 포맷
  • Webhook·REST API (CI 연동)
